KingSejong
Unreal Engine 5.6 C++ Team 3인

조선 시대를 배경으로 한 온라인 멀티플레이어 미니게임 모음입니다. 플레이어들은 로비에서 방을 만들고 달리기 · 토론 · 배틀 세 가지 미니게임 중 하나를 선택해 함께 플레이합니다.

본인 담당 파트 : 세션 시스템 전반 · 로비 UI · 달리기 미니게임 · 캐릭터 선택

1. 멀티플레이어 세션 시스템
  • OnlineSubsystem을 직접 연동하여 세션 생성 · 탐색 · 접속 · 파괴 전 과정 구현
  • Steam / LAN 환경 자동 감지 및 분기 처리
  • Steam 서버에 한글 방 이름을 저장하기 위한 Base64 인코딩 / 디코딩 적용
  • 서버 RPC → Multicast RPC → Client RPC 구조로 세션 퇴장 시 모든 플레이어 동시 로비 복귀
  • 네트워크 장애(서버 종료) 시 OnNetworkFailure 콜백으로 클라이언트 자동 복귀
// 한글 방 이름을 Steam 서버에 안전하게 저장하기 위한 Base64 인코딩
FString roomname = StringBase64Encode(RoomName);
SessionSettings.Set<FString>(FName("Room_Name"), roomname,
    EOnlineDataAdvertisementType::ViaOnlineServiceAndPing);
2. 로비 UI
  • WidgetSwitcher로 방 만들기 / 방 찾기 화면 전환
  • 게임 종류(달리기 · 토론 · 배틀) 체크박스 — 하나만 선택되도록 상호 배타 처리
  • 슬라이더로 최대 인원 실시간 설정
  • 세션 탐색 결과를 델리게이트로 수신 후 스크롤박스에 슬롯 위젯 동적 생성
  • 방 미선택 상태에서 입장 버튼 클릭 시 크래시 방지 처리
3. 달리기 미니게임
  • 결승선(FinishLine) — OnComponentBeginOverlap으로 통과 감지, 게임모드에 순서 전달
  • 결승선 통과 순서대로 초성 퀴즈 기회 부여, 오답 시 다음 플레이어로 자동 전환
  • 퀴즈 답변 중 나머지 플레이어는 답변자 폰을 자동 관전하도록 전환
  • DataTable에서 랜덤 퀴즈 선택 후 Multicast RPC로 모든 클라이언트에 동기화
  • RPC로 특정 타이밍에 플레이어 입력 활성화 / 비활성화 제어
  • 승리 / 패배 몽타주 재생 및 엔드게임 UI 전환

4. 캐릭터 선택 시스템
  • 이전 / 다음 버튼으로 스켈레탈 메시 실시간 교체 및 프리뷰
  • 선택 확정 시 메시 인덱스를 GameInstance에 저장해 게임 내내 유지
  • 선택 완료 후 마우스 숨김 및 게임 입력 모드로 전환

Isekiro
Unreal Engine 5.6 C++ 개인 프로젝트

세키로(Sekiro: Shadows Die Twice) 스타일의 3인칭 액션 게임입니다. 패리 · 자세 게이지 · 처형 등 세키로의 핵심 전투 메커니즘을 언리얼 엔진 C++로 직접 구현했습니다.

시연 영상
1. 보스 AI — 컴포넌트 기반 FSM
  • UFSMComponent로 보스 행동 상태를 관리하는 핵심 컴포넌트 설계
  • 각 상태를 UStateObject 클래스로 분리 — Start / End / Interrupt 인터페이스 통일
  • 플레이어와의 거리에 따라 근거리(NEAR) / 원거리(FAR) 공간 분리 후 상태 선택
  • 공격받을수록 회피(Dodge) 확률이 점진적으로 증가하는 적응형 AI
  • Block / Parry 확률을 상태별로 독립 설정 가능
카테고리상태
반응 (Stun)Hit, Block, Parry, Deflected
근거리 공격NormalAttack, CounterAttack, Strafe
원거리 공격DistanceAttack, ThrustAttack, Run, PatternAttack, LungeAttack
전환Dodge, DodgeAttack, Jump, JumpStrike, JumpFire
기타Death
2. 타겟 락온 시스템
  • UActorComponent 기반 모듈형 설계 — 모든 캐릭터에 즉시 탑재 가능
  • 내적(Dot Product)으로 카메라 정면 벡터와 타겟 벡터 유사도 측정, 화면 중앙에 가장 가까운 적 우선 선택
  • RInterp To로 카메라 회전 부드럽게 보간, 락온 상태에서 Strafing 이동 자동 전환
  • Line Trace로 장애물 뒤의 적은 타겟 대상에서 자동 제외
3. 그래플링 훅 시스템
  • 타겟 지점 감지 후 하단 트레이스로 실제 착지 가능 지형(Walkable Surface) 사전 검증
  • Timeline + Curve로 발사 가속 / 도착 감속을 조합한 물리적으로 자연스러운 이동 궤적 구현
  • Cable Component로 발사체와 캐릭터 손 소켓 사이의 물리 연결 및 환경 충돌 구현
  • 애니메이션 노티파이(Anim Notify)로 이동 시작 · 종료 시점과 Flying / Falling 상태 정밀 동기화
4. 패링 시스템
  • USphereComponent 기반 URealParryBox의 Overlap으로 패리 판정 — 일반 약 8프레임, 위험(Hazard) 상태 1프레임으로 난이도 연동
  • 패리 입력 시 데미지 채널 충돌 응답을 즉시 Ignore로 전환하여 무적 판정 구현, 패리 종료 시 복구
  • 패리 성공 시 SetGlobalTimeDilation(0.85f)로 슬로우 모션 적용 후 0.5초 뒤 타이머로 복구
  • 패리 성공마다 보스 자세 데미지(3) 누적, 자세 붕괴 시 처형 연출 트리거
  • 패리 윈도우와 쿨다운을 별도 FTimerHandle로 분리 — 연속 패리 어뷰징 방지

VirtualIdol
Unreal Engine 5 C++ Blueprint Team 3인

VRM 모델을 활용한 버튜버 온라인 콘서트 플랫폼입니다. 관객들이 가상 무대에 모여 버튜버의 공연을 감상하고, 팬들이 직접 무대 환경을 커스터마이징하여 세션을 개설할 수 있습니다.

본인 담당 파트 : 무대 액터 배치 시스템 · 버튜버 모델링 적용 · 무대 설정 UI 구현 · RemoCap UDP/OSC 본 데이터 손실 초기화

1. 무대 액터 배치 시스템
  • 하늘(Sky) · 테마(Theme) · 지면(Floor) · 이펙트(VFX) 4가지 카테고리를 독립 조합하는 모듈형 무대 구성
  • 테마 선택 시 StreamLevel 동적 로드/언로드 — 이전 레벨을 0.2초 간격 지연 타이머로 순차 언로드하여 비동기 충돌 방지
  • 액터 태그(VFX · Floor · Skybox · Theme) 기반 일괄 Destroy 후 재스폰으로 깔끔한 교체 처리
  • SceneCaptureComponent2D + UTextureRenderTarget2D로 현재 카메라 시점 실시간 캡처하여 썸네일 생성
  • 서버/클라이언트 분기 처리 — CreateDummyStage()로 클라이언트 프리뷰 동기화 제공
// 스트리밍 레벨 교체 시 이전 레벨을 지연 언로드하여 비동기 충돌 방지
float UnloadDelay = 0.4f;
for (const FName& LevelName : LevelsToUnload)
{
    GetWorld()->GetTimerManager().SetTimer(UnloadTimerHandle,
        [this, LevelName]() {
            UGameplayStatics::UnloadStreamLevel(GetWorld(), LevelName, ...);
        }, UnloadDelay, false);
    UnloadDelay += 0.2f;
}
2. 버튜버 모델링 적용
  • VRM4U 플러그인을 통해 VRM 형식의 버튜버 모델을 언리얼 엔진에 임포트 및 적용
  • 게임 플레이 중 스켈레탈 메시를 런타임에 동적 교체하여 공연 도중 의상 및 모델 전환 구현
  • Morph Target 제어를 애니메이션과 연동하여 실시간 표정 변화 반영
  • Mocopi 기반 전신 모션 캡처 데이터를 UDP 소켓으로 수신하여 VRM 모델 뼈대에 적용
3. RemoCap UDP/OSC 본 데이터 손실 초기화
  • RemoCap 앱의 본(Bone) 데이터 전송 방식을 분석하여 기존 로직의 데이터 손실 원인 파악
  • UDP 기반 OSC 프로토콜로 수신 구조 수정 — FUdpSocketBuilder로 포트 바인딩 후 FUdpSocketReceiver 리스너 스레드 구성
  • 패킷 수신 누락 시 이전 프레임 데이터를 유지하던 문제를 초기 상태 리셋 로직으로 교체하여 포즈 고착 현상 제거
  • 100ms 폴링 간격 백그라운드 스레드에서 연속 수신 처리, 메인 스레드와 안전 동기화
4. 무대 설정 UI
  • 날씨 · 테마 · 이펙트 · 지면 4개 카테고리 탭 — 탭 전환 시 해당 옵션 패널만 표시
  • 선택된 카테고리 버튼에 Scale 1.1 렌더 트랜스폼 적용, 이전 버튼 원상 복귀로 현재 선택 상태 시각화
  • Border 배열로 선택된 항목에만 SelectedBox 텍스처를 적용하는 하이라이트 시스템
  • WidgetSwitcher로 설정 → 캡처 확인 → 최종 확인 3단계 플로우 전환
  • 0.3초 지연 타이머로 UI 버튼 은닉 후 씬 캡처 — 버튼이 썸네일에 찍히는 문제 해결
  • 스테이지 이름 입력 후 썸네일 + 설정 정보를 HTTP Multipart로 서버 업로드
